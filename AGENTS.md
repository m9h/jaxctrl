## Project Identity

- **Name**: jaxctrl
- **Description**: Differentiable control theory in JAX: system identification, Lyapunov, Riccati, Gramians, tensor control, and hypergraph controllability.
- **Language**: Python
- **Build system**: hatchling
- **Package manager**: uv
- **License**: Apache-2.0


## Quick Start

```bash
cd path/to/jaxctrl
uv sync
uv run pytest
```

## Module Map

| Module | Purpose | Public API |
|--------|---------|------------|
| `jaxctrl/__init__.py` | Top-level package: re-exports all public symbols across the four layers. | `__version__` |
| `jaxctrl/_sysid.py` | Layer 0 — data-driven system identification. | `SINDyOptimizer`, `KoopmanEstimator`, `polynomial_library`, `fourier_library` |
| `jaxctrl/_lyapunov.py` | Layer 1 — continuous and discrete Lyapunov solvers. | `solve_continuous_lyapunov`, `solve_discrete_lyapunov`, `is_stable`, `is_schur_stable` |
| `jaxctrl/_riccati.py` | Layer 1 — continuous/discrete algebraic Riccati equations and LQR. | `solve_continuous_are`, `solve_discrete_are`, `lqr`, `dlqr` |
| `jaxctrl/_gramian.py` | Layer 1 — controllability and observability matrices and Gramians. | `controllability_matrix`, `observability_matrix`, `controllability_gramian`, `observability_gramian` |
| `jaxctrl/_controllability.py` | Layer 1 — controllability/observability/stabilizability/detectability tests and minimum-energy control. | `is_controllable`, `is_observable`, `is_stabilizable`, `is_detectable`, `minimum_energy` |
| `jaxctrl/_simulate.py` | Layer 1 — closed- and open-loop LTI simulation (Diffrax adaptive ODE or matrix-exponential fallback). | `simulate_lti`, `simulate_closed_loop` |
| `jaxctrl/_tensor_ops.py` | Layer 2 — tensor algebra primitives (unfold/fold, contractions, HOSVD, Tucker). | `tensor_unfold`, `tensor_fold`, `einstein_product`, `tensor_contract`, `mode_dot`, `hosvd`, `tucker_to_tensor`, `khatri_rao`, `tensor_trace`, `symmetrize_tensor` |
| `jaxctrl/_tensor_eigen.py` | Layer 2 — Z- and H-eigenvalue iteration for symmetric tensors. | `z_eigenvalues`, `h_eigenvalues`, `spectral_radius`, `tensor_power_method` |
| `jaxctrl/_arte.py` | Layer 2 — Algebraic Riccati Tensor Equation, tensor Lyapunov, multilinear LQR. | `solve_arte`, `tensor_lyapunov`, `multilinear_lqr` |
| `jaxctrl/_hypergraph_control.py` | Layer 3 — hypergraph controllability (requires `hgx`). | `HypergraphControlSystem`, `adjacency_tensor`, `laplacian_tensor`, `tensor_kalman_rank`, `minimum_driver_nodes`, `control_energy`, `controllability_profile`, `hypergraph_linear_system` |


## Testing

- **Framework**: pytest
- **Test location**: test/
- **Run**: `uv run pytest test/ -v`
- **No mocks**: Tests use real implementations, not unittest.mock.
- **Hypergraph tests**: `test/test_hypergraph_control.py` requires the `m9h/hgx` package; the PyPI `hgx` is a different project. CI excludes this file via `--ignore=test/test_hypergraph_control.py`.

## Dependencies

|-------|----------|---------|

| `hypergraph` | hgx | Layer 3 hypergraph controllability stack |

| `tests` | pytest, beartype | Test runner and runtime type checks |
